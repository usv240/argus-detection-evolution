# ADR-0005: Human approval and honest reporting

**Status:** Accepted
**Date:** 2026-06-05

---

By the end of a run, ARGUS has produced an evolved detection rule that measurably catches more
than the one it started with. The simplest next step would be to just turn it on in Splunk.

We didn't do that, on purpose. A detection rule decides what does or doesn't raise a security
alert for a real team. A rule that was rewritten by an AI, and tested against attack variants that
were also generated by an AI (even though they're grounded in real data patterns), is exactly the
kind of change that should have a person look at it before it goes live.

So the evolved rule is shown next to the original, with the parts that changed highlighted, and
Blue's plain-English explanation of what changed and why. The user can Approve, Edit, or Reject.
Even after Approve, the resulting Splunk saved search is created switched off (`disabled=1`).
Turning it on is a separate, deliberate action inside Splunk itself, not something ARGUS does for
you. Exporting the rule as an installable app follows the same pattern: it ships disabled, and
includes a real validation report from Splunk's own AppInspect tool, so whoever installs it can
see it was checked.

The same kind of honesty applies to the results, not just the deploy step. At the end of a run,
ARGUS lists any attack variants that the final rule still doesn't catch. We call this the residual
frontier. It would be easy to only show the wins, the variants that are now caught, and quietly
drop the rest. Instead ARGUS shows the gaps too, each with its own anomaly score, because "here's
what's still a gap, and here's how suspicious it would have looked anyway" is more useful to a
security team than a claim of "done."

The honest tradeoff is that some runs end below 100% coverage, and ARGUS says so plainly. That's a
less satisfying result than always reporting a perfect score. But a tool that always claims to be
finished isn't one you can trust, and a rule-changing tool that won't ship without a human's
sign-off is the kind a security team could actually turn on.
